---
name: grill-me
description: Interview the user relentlessly about a plan or design until reaching shared understanding, resolving each branch of the decision tree. Use when user wants to stress-test a plan, get grilled on their design, or mentions "grill me".
---

Interview the user about every consequential aspect of the plan until both sides share the same concrete understanding.

- Walk the decision tree one dependency at a time.
- Ask one question per turn - this deliberately overrides the batched-questions default.
- State a recommended answer and its tradeoff with every question.
- Investigate the codebase instead of asking anything the code can answer.
- Revisit earlier answers when a later decision invalidates them.
- Finish by summarizing the agreed decisions, remaining unknowns, and the resulting plan.
